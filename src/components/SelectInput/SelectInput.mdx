# SelectInput

Dropdown that lets the user choose one option from a predefined list. Part of the Forms family alongside TextInput, TextareaInput, and NumberInput.

<Canvas of={SelectInputStories.AllStates} />

---

## When to use

- **SelectInput** — user must pick from a known, bounded set of options (status, country, session type)
- **RadioGroup** — when there are ≤ 4 options and you want them all visible at once
- **OptionList** — when options are rich (icons, descriptions, sections, multi-select)

Do not use SelectInput for open-ended input — use TextInput instead.

## CSS selector structure

SelectInput's styling is primarily applied to the native `<select>` element to keep it accessible and avoid custom dropdown implementations.

```css
@import 'tokens/tokens.css';
```

```html
<!-- Basic -->
<div style="width: 100%;">
  <label style="font-size: 13px; font-weight: 500; color: #303030; display: block; margin-bottom: 6px;">
    Country
  </label>
  <select class="nx-select">
    <option value="" disabled selected>Select a country</option>
    <option value="us">United States</option>
    <option value="ca">Canada</option>
  </select>
</div>
```

```css
.nx-select {
  width: 100%;
  padding: 10px 40px 10px 14px;
  border-radius: var(--nx-radius-sm);        /* 8px */
  border: 1.5px solid var(--nx-border-input); /* #8a8a8a */
  background: var(--nx-bg-input);             /* #fdfdfd */
  font-family: var(--nx-font-family);
  font-size: var(--nx-font-size-body);        /* 14px */
  color: var(--nx-text-default);
  outline: none;
  cursor: pointer;
  appearance: none;
  -webkit-appearance: none;
  /* Chevron via inline SVG data URI */
  background-image: url("data:image/svg+xml,...");
  background-repeat: no-repeat;
  background-position: right 12px center;
}

  box-shadow: var(--nx-focus-ring);
```

## Accessibility

- Wrap in a `<label>` or use `aria-labelledby` — never use placeholder as the sole label
- The placeholder `<option>` should have `disabled` and `value=""` so it cannot be re-selected
- Required fields use the native `required` attribute on `<select>`
- Custom chevron is decorative — `aria-hidden="true"` if implemented as a separate element
- Native `<select>` has full keyboard and screen-reader support out of the box — do not replace it with a `<div>` dropdown unless OptionList's richer feature set is needed

## React component

```jsx
import { SelectInput } from 'components/SelectInput/SelectInput.jsx';

const OPTIONS = ['Morning', 'Evening', 'Both'];

// Basic
<SelectInput
  label="Session type"
  placeholder="Select type"
  options={OPTIONS}
  value={val}
  onChange={e => setVal(e.target.value)}
/>

// Required
<SelectInput
  label="Status"
  required
  placeholder="Choose status"
  options={['Active', 'Inactive', 'Pending']}
  value={val}
  onChange={fn}
/>

// Disabled
<SelectInput label="Region" options={['North']} disabled />
```
