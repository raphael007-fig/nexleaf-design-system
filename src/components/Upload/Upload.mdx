# Upload

A file-upload field with three per-file states — **uploading**, **done**, and **error** — plus an "Add file" trigger and an optional helper line that documents the constraints (file types, count, size).

<Canvas of={UploadStories.ThreeStates} />

---

## When to use

- A form needs one or more documents/images attached (e.g. compliance evidence, vaccine cold-chain records).
- The set of allowed types and the per-file size limit are well-defined and worth stating up front in the helper line.

Do **not** use Upload for bulk dataset imports — those need a dedicated drag‑drop region with chunked progress UI.

---

## Anatomy

| Region | Notes |
|---|---|
| **Label** | Inline form label. Pass `required` for a red asterisk. |
| **Add file button** | Design-system `<Btn variant="secondary">` with an upload-arrow icon. Triggers the hidden `<input type="file">`. Disables itself automatically when `files.length >= maxFiles`. |
| **Helper text** | Single-line copy describing constraints. Use it to set expectations (allowed types, max count, max size). |
| **File rows** | One row per file. The right-side action and the left badge swap based on `file.status`. |

---

## File states

| `status` | Left badge | Right action |
|---|---|---|
| `uploading` | Coloured file-type tile (PDF/JPG/…) | Circular progress ring with `%` + plain "X" to cancel |
| `done` | Coloured file-type tile | Filled red square icon button to delete |
| `error` | Pink circle with warning triangle | `<Btn variant="ghost">` "Try Again" link |

The file-type tile colour is derived from the file extension:

| Extension | Colour |
|---|---|
| `pdf` | `#e51c00` (Btn critical red) |
| `jpg` / `jpeg` / `png` / `gif` / `webp` / `svg` | `#16a34a` |
| `doc` / `docx` / `txt` | `#2563eb` |
| `xls` / `xlsx` / `csv` | `#0e7340` |
| `zip` / `rar` / `7z` | `#a16207` |
| anything else | `#616161` (text-subdued) |

## Controlled API

Upload is **stateless**. The consumer owns the `files` array and updates it in response to callbacks. The component never mutates files internally — it just renders the current state and emits events.

Each file is shaped as:

```js
{
  id:           string,                          // stable id
  name:         string,                          // filename including extension
  size:         number,                          // bytes
  uploadedAt:   Date | string | null,            // shown in the meta row
  status:       'uploading' | 'done' | 'error',
  progress?:    number,                          // 0–100 (uploading only)
  errorMessage?: string                          // shown in red (error only)
}
```

Callbacks:

| Prop | Fires when | Consumer should |
|---|---|---|
| `onAddFiles(File[])` | User picks one or more files from the OS picker. | Convert each `File` into the file shape above, push onto `files`, and start the upload. |
| `onCancel(id)` | User clicks the X next to an uploading row. | Abort the in-flight upload and remove the file from `files`. |
| `onRemove(id)` | User clicks the red delete button on a `done` row. | Remove the file from `files` (and from the server, if needed). |
| `onRetry(id)` | User clicks "Try Again" on an `error` row. | Reset the file to `status: 'uploading', progress: 0` and re-upload. |

## Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `label` | `string` | `"Upload document"` | Field label rendered above the button. |
| `required` | `boolean` | `false` | Adds a red asterisk to the label. |
| `helperText` | `string` | `"Upload up to 5 files (JPEG, PNG), max 10 MB each."` | Constraints line under the button. Pass `null` / `""` to hide. |
| `accept` | `string` | `"image/jpeg,image/png,application/pdf"` | Forwarded to the native `<input type="file" accept>`. |
| `multiple` | `boolean` | `true` | Whether the picker allows selecting multiple files at once. |
| `maxFiles` | `number` | `5` | When `files.length >= maxFiles`, "Add file" disables. |
| `files` | `File[]` (see shape) | `[]` | Current set of files. |
| `onAddFiles` | `(File[]) => void` | — | Fires with the native `File` objects after the picker closes. |
| `onCancel` | `(id) => void` | — | Cancel an in-flight upload. |
| `onRemove` | `(id) => void` | — | Delete a completed file. |
| `onRetry` | `(id) => void` | — | Retry a failed upload. |
| `disabled` | `boolean` | `false` | Disables the whole component — picker won't open and per-file actions are inert. |

## Accessibility

- The visible "Add file" button delegates to a visually-hidden but focusable `<input type="file">`. Both share the label via `htmlFor`/`id`.
- Per-row actions all carry `aria-label` referencing the filename (e.g. *"Cancel upload of Name of document.pdf"*).
- The progress ring is decorative (`aria-hidden`); consumers wanting screen-reader progress announcements should include their own `aria-live` region tracking the upload state.
- Disabled state uses `cursor: not-allowed` and the design-system `#b5b5b5` text colour throughout.
- The **Add file** button auto-disables when `files.length >= maxFiles` (default 5), so users can't exceed the documented limit. Once a file is removed (or upload cancelled), the button re-enables.

## Angular component

```ts
import { NxUploadModule } from 'components/Upload/upload.module';

<nx-upload
  label="Upload document"
  [required]="true"
  helperText="Upload up to 5 files (JPEG, PNG), max 10 MB each."
  accept="image/jpeg,image/png,application/pdf"
  [multiple]="true"
  [maxFiles]="5"
  [files]="files"
  (addFiles)="onAdd($event)"
  (cancel)="onCancel($event)"
  (remove)="onRemove($event)"
  (retry)="onRetry($event)"
/>

// Disabled (whole component)
<nx-upload [disabled]="true" [files]="files" label="Upload document" />
```

File model (TypeScript):

```ts
interface NxUploadFile {
  id: string;
  name: string;
  size: number;                                  // bytes
  uploadedAt?: Date | string | null;             // shown in meta row
  status: 'uploading' | 'done' | 'error';
  progress?: number;                             // 0-100 for uploading
  errorMessage?: string;                         // shown in red for error
}
```

## React component

```jsx
import { Upload } from 'components/Upload/Upload.jsx';

const [files, setFiles] = useState([]);

<Upload
  label="Upload document"
  required
  helperText="Upload up to 5 files (JPEG, PNG), max 10 MB each."
  accept="image/jpeg,image/png,application/pdf"
  maxFiles={5}
  files={files}
  onAddFiles={(picked) => {
    setFiles((curr) => [
      ...curr,
      ...picked.map((p, i) => ({
        id: `${Date.now()}-${i}`,
        name: p.name,
        size: p.size,
        uploadedAt: new Date(),
        status: 'uploading',
        progress: 0,
      })),
    ]);
  }}
  onCancel={(id) => setFiles((c) => c.filter((f) => f.id !== id))}
  onRemove={(id) => setFiles((c) => c.filter((f) => f.id !== id))}
  onRetry={(id) =>
    setFiles((c) => c.map((f) =>
      f.id === id ? { ...f, status: 'uploading', progress: 0, errorMessage: undefined } : f
    ))
  }
/>

// Disabled (whole component)
<Upload disabled files={files} label="Upload document" />
```
