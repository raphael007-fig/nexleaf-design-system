# Badge

Compact status label. Communicates category, state, or count at a glance. Never interactive — badges are purely decorative/informational.

## When to use

- Status labels on list rows (Pending, Completed, Locked)
- Metadata tags on page headers
- Count indicators on tabs or list items
- Progress indicators (incomplete / partial / complete)

Do **not** use a badge for an action — use a `Btn` instead.

## Tone palette

### Regular (tinted)

| Tone | Background | Text | Use for |
|------|-----------|------|---------|
| `default` | `rgba(0,0,0,0.06)` | `#616161` | Neutral state, enabled, new |
| `info` | `#e0f0ff` | `#00527c` | Informational metadata |
| `success` | `#cdfee1` | `#0c5132` | Completed, active, healthy |
| `attention` | `#ffef9d` | `#4f4700` | Pending, needs review |
| `warning` | `#ffd6a4` | `#5e4200` | At risk, expiring soon |
| `critical` | `#fedad9` | `#8e1f0b` | Error, failed, overdue |
| `read-only` | transparent | `#616161` | Read-only annotations |

### Strong (filled)

Use Strong badges when higher visual contrast is required — e.g., on busy backgrounds, dashboards, or next to other strong-tone badges.

| Tone | Background | Text | Use for |
|------|-----------|------|---------|
| `info-strong` | `#91d0ff` | `#00527c` | Highlighted informational metadata |
| `success-strong` | `#29845a` | `#f8fffb` | Confirmed active / healthy |
| `attention-strong` | `#ffe600` | `#332e00` | Urgent pending action |
| `warning-strong` | `#ffb800` | `#5e4200` | At-risk, expiring soon |
| `critical-strong` | `#e51c00` | `#fffbfb` | Error, failed, critical alert |

## Icons & dismiss

Badges accept three optional slots around the label:

| Prop | Type | Renders |
|------|------|---------|
| `progressIndicator` | boolean | Filled 8×8px rounded-square dot in the tone's dot color (left of label) |
| `icon` | boolean | Info-circle SVG icon (left of label) |
| `onDismiss` | function | × dismiss button (right of label) |

<Canvas of={BadgeStories.AllStates} />

## Progress dots

Three progress variants are shown as a small shape to the left of the label:

| `progress` value | Shape | Fill |
|-----------------|-------|------|
| `incomplete` | Hollow circle | none |
| `partial` | Rounded square | semi-filled (40% opacity) |
| `complete` | Rounded square | solid `#616161` |

## Accessibility

- Badges are `<span>` elements — purely presentational
- When a badge communicates status critical to a decision, supplement with visible text or an `aria-label` on the containing row
- Never use color alone to convey meaning — the text label must be present
- Dismiss buttons must have `aria-label="Dismiss"` on the `<button>` element

```jsx
import { Badge, StatusBadge, IconInfoCircle } from 'components/Badge/Badge.jsx';

// Regular tones
<Badge tone="success">Completed</Badge>
<Badge tone="attention">Pending</Badge>
<Badge tone="critical" size="large">Overdue</Badge>
<Badge tone="default" progress="partial">In progress</Badge>

// Strong tones
<Badge tone="info-strong">Info</Badge>
<Badge tone="success-strong">Active</Badge>
<Badge tone="warning-strong">Warning</Badge>
<Badge tone="attention-strong">Attention</Badge>
<Badge tone="critical-strong" size="large">Failed</Badge>

// With progress indicator dot, icon, or dismiss
<Badge tone="info-strong" progressIndicator>Label</Badge>
<Badge tone="info-strong" icon>Label</Badge>
<Badge tone="info-strong" onDismiss={() => handleDismiss()}>Label</Badge>

// StatusBadge — maps semantic status strings to tones automatically
<StatusBadge status="pending" />    // → attention tone
<StatusBadge status="completed" />  // → success tone
<StatusBadge status="locked" />     // → default tone
```
