# Card

Structured content containers for displaying equipment details, location context, contact information, and associated component lists. Cards use the shadow-100 border treatment and an `8px` border radius throughout.

<Canvas of={CardStories.AllLayouts} />

---

## When to use

- Equipment or asset detail pages where data is grouped into sections
- Dashboard sidebars showing location, contact, or status context
- Anywhere tabular sub-data needs to be collapsible (Layout Type 2)

## Layout Type 1 — Main Detail Card

<Canvas of={CardStories.LayoutType1} />

Equipment detail card with three sections:

2. **Fields** — 2-column key/value grid, minimum row height `44px`
3. **Notes** — free-text block with `CheckCircleIcon` section title

## Layout Type 2 — Accordion Table

<Canvas of={CardStories.LayoutType2} />

Collapsible card revealing a tabbed component table. The accordion header shows a title + optional description. The body contains:
- **Tabs** + a sort icon button
- A compact data table with 4 columns and a View link per row
- Optional table `Pagination` at the bottom

## Layout Type 5 — QR Code Card

<Canvas of={CardStories.LayoutType5} />

QR Code card with two states driven by the `qrCodeSrc` prop:

- **Empty** (`qrCodeSrc` is `null` / omitted) — a 48×48 QR icon inside an 88×88 soft-blue circle, a centered "No QR Code Assigned" caption, and an **Assign QR Code** secondary button.
- **Assigned** (`qrCodeSrc` provided) — renders the scannable QR image at 160×160 with a **View QR Code** secondary button. Clicking the button opens a full-size preview in the **same `Overlay` modal layout** used by the Page header image preview — so both interactions feel identical (Esc, backdrop click, and the × button all close).

```jsx
// Empty state
<CardLayoutType5 title="QR Code" onAssign={() => openAssignFlow()} />

// Assigned state — clicking "View QR Code" opens the modal automatically
<CardLayoutType5
  title="QR Code"
  qrCodeSrc="/equipment/qr/EQ-10022.png"
  onView={() => trackQrPreviewOpened()}   // optional analytics hook
/>

// Loading
<CardLayoutType5 title="QR Code" loading />
```

All copy is overridable via `emptyText`, `emptyActionLabel`, and `assignedActionLabel` for localisation.

## Layout Type 6 — Header / Action Card

<Canvas of={CardStories.LayoutType6} />

A card with a header strip and a free-form body. The header pairs a leading **icon + title + badge** on the left with an optional **action button** on the top right; the body is whatever you pass as `children` — typically a row (web) or stack (mobile) of [`Cell`](/docs/components-lists-cell-docs--docs) rows.

Set `tone="critical"` to tint the card edge red (the Figma "Immediate Action" alert card). The header badge defaults to the card tone — `critical` → critical badge, otherwise a default-tone badge — and can be overridden with `badgeTone`.

```jsx
// Critical alert card — web (cells in a row), mobile (cells stacked)
<CardLayoutType6
  tone="critical"
  icon={<AlertTriangleIcon />}
  title="Action Required"
  badge="5 Urgent Issues"
  actionLabel="View All Issues"
  onAction={() => goTo('/issues')}
>
  <div style={{ display: 'flex', gap: 12 }}>{/* flexDirection: 'column' on mobile */}
    <div style={{ flex: 1, minWidth: 0 }}>
      <Cell icon={<GaugeIcon />} iconTone="critical"
        title="Temperature exceeds threshold"
        description="Incubator HC 1501-2023-001 · Main Laboratory"
        hasChevron onClick={() => goTo('/equipment/HC-1501')} />
    </div>
    <div style={{ flex: 1, minWidth: 0 }}>
      <Cell icon={<WrenchIcon />} iconTone="warning"
        title="Maintenance due"
        description="Generator GEN-2023-005 · Power Room"
        hasChevron onClick={() => goTo('/equipment/GEN-2005')} />
    </div>
  </div>
</CardLayoutType6>
```

The web/mobile arrangement can be handled for you. Set **`stackOnMobile`** and pass the cells directly — the card lays them out as equal-width columns on wide screens and stacks them full-width at/below the `sm` breakpoint (640px), via the shared `useIsMobile` hook. No manual flex wrapper needed:

```jsx
<CardLayoutType6
  tone="critical"
  icon={<AlertTriangleIcon />}
  title="Action Required"
  badge="5 Urgent Issues"
  actionLabel="View All Issues"
  onAction={() => goTo('/issues')}
  stackOnMobile
>
  <Cell icon={<GaugeIcon />} iconTone="critical"
    title="Temperature exceeds threshold"
    description="Incubator HC 1501-2023-001 · Main Laboratory"
    hasChevron onClick={() => goTo('/equipment/HC-1501')} />
  <Cell icon={<WrenchIcon />} iconTone="warning"
    title="Maintenance due"
    description="Generator GEN-2023-005 · Power Room"
    hasChevron onClick={() => goTo('/equipment/GEN-2005')} />
</CardLayoutType6>
```

<Canvas of={CardStories.LayoutType6Responsive} />

Leave `stackOnMobile` off (the default) when you need full control over the body arrangement — then wrap the children yourself, switching `flexDirection` from `row` (web) to `column` (mobile).

### Props

| Prop | Type | Default | Notes |
|------|------|---------|-------|
| `icon` | node | — | 20×20 SVG before the title. |
| `title` | string | — | Header title (12px / 550). |
| `badge` | node / string | — | Header badge. String → `<Badge>` using `badgeTone`. |
| `badgeTone` | Badge tone key | `'critical'` when `tone='critical'`, else `'default'` | Tint of the string badge. |
| `actionLabel` | string | — | Renders a top-right ghost (plain) button when set. |
| `onAction` | fn | — | Action button click. |
| `tone` | `'default' \| 'critical'` | `'default'` | Tints the card edge (critical = red bevel). |
| `loading` | bool | `false` | Render a header + `Cell`-row skeleton instead of content. |
| `loadingRows` | number | `2` | `Cell` skeleton rows to show when `loading`. |
| `stackOnMobile` | bool | `false` | When set, lays children out as equal columns on wide screens and stacks them full-width at/below `sm` (640px). |
| `bodyGap` | number | `12` | Gap between body children when `stackOnMobile` is set. |
| `children` | node | — | Body content — e.g. a flex row/column of `Cell`s. |

When `loading`, the card keeps its tinted surface and renders a header placeholder (icon + title + badge + action) over `loadingRows` stacked `Cell` skeletons — so the alert card holds its shape until the issues data arrives.

## Anatomy

### Card container
| Property | Value |
|----------|-------|
| Background | `#ffffff` |
| Border radius | `8px` |
| Padding | `12px` |
| Gap (children) | `12px` |
| Shadow | shadow-100 (see below) |

### Shadow (shadow-100)
```
0 1px 0 rgba(26,26,26,0.07),
inset 1px 0 0 rgba(0,0,0,0.13),
inset -1px 0 0 rgba(0,0,0,0.13),
inset 0 -1px 0 rgba(0,0,0,0.17),
inset 0 1px 0 rgba(204,204,204,0.5)
```

### CardField
| Part | Size | Weight | Color |
|------|------|--------|-------|
| Label | 13px | 650 | `#303030` |
| Value (no icon) | 13px | 450 | `#303030` |
| Value (with icon) | 13px | 450 | `#616161` |
| Value (link) | 13px | 450 | `#005bd3`, underlined |
| Icon | 20×20px | — | `#616161` |
| Min row height | 44px | — | — |

### CardImages
| Property | Value |
|----------|-------|
| Thumbnail size | 120×120px |
| Border radius | `8px` |
| Gap | `16px` |
