# Modal

An accessible dialog primitive. Modal wraps content in a focus-trapped, scroll-locked, ARIA-correct dialog rendered through a portal on top of a dimmed scrim. Every blocking surface in the system (image zoom, confirmations, AI panels) should sit on top of this instead of re-implementing backdrop + Escape handling by hand.

<Canvas of={ModalStories.Sizes} />

---

## When to use

- **Confirmations** — destructive or terms-style decisions that must block the page until resolved.
- **Focused tasks** — a short form or single decision that shouldn't navigate the user away.
- **Mobile takeovers** — the `fullscreen` size fills the viewport for small screens.

Avoid stacking modals. For non-blocking, lightweight surfaces anchored to a trigger, prefer `Popover` instead.

---

## Two usage modes

Modal supports two rendering modes, chosen automatically:

| Mode | Trigger | What Modal renders |
|------|---------|--------------------|
| **Structured** | `title` and/or `footer` passed | White card surface, gray header with close button, scrollable body, footer action row. |
| **Raw** | only `children` | Just the focus-trapped dialog wrapper; children supply their own surface. Provide `ariaLabel` for an accessible name. |

```jsx
// Structured
<Modal open={open} onClose={close} title="Edit reading" footer={<Btn>Save</Btn>}>
  …form…
</Modal>

// Raw (e.g. image zoom) — children own the surface
<Modal open={open} onClose={close} ariaLabel="Image preview" showCloseButton={false}>
  <img src={src} alt="" />
</Modal>
```

## Sizes

<Canvas of={ModalStories.Sizes} />

Structured modals resolve their width from the `size` token. `fullscreen` fills the viewport with square corners and no backdrop inset — reserve it for mobile, per the design spec.

| `size` | Width | Use |
|--------|-------|-----|
| `small` | `380px` | Compact confirmations, single decisions. |
| `large` | `620px` (default) | Standard forms and content. |
| `fullscreen` | fills viewport | Mobile takeovers; square corners, no shadow. |

An explicit `maxWidth` always wins over the size token, so one-off widths remain possible without breaking existing call-sites.

## Toggles — title & actions

<Canvas of={ModalStories.WithoutTitle} />

With no `title`, the header collapses to just the floating close button (top-right). Pass `showCloseButton={false}` to drop it entirely (provide an `ariaLabel` so the dialog still has a name).

<Canvas of={ModalStories.WithoutSecondaryAction} />

The footer is a **slot** — the consumer supplies the action buttons. A single-action modal simply passes one primary `Btn`; a two-action modal passes a secondary + primary pair. This keeps the action conventions in the consumer's hands rather than baked into Modal.

---

## Interaction states

- **Open** — entrance uses the shared `nxFadeInScale` keyframe (`0.18s ease-out`); skipped when the user prefers reduced motion.
- **Focus** — focus is trapped within the dialog (`useFocusTrap`); `Tab`/`Shift+Tab` cycle inside. On close, focus returns to the element that opened the modal (`useReturnFocus`).
- **Escape** — closes by default (`closeOnEscape`). The close button shows the brand focus ring (`#005bd3`) when keyboard-focused.
- **Backdrop** — clicking the scrim closes by default (`closeOnBackdrop`). The check is on `mousedown` against the backdrop element itself, so a drag that starts inside the dialog won't accidentally dismiss it.
- **Scroll** — background scroll is locked while open (`useScrollLock`); the body region scrolls internally when content overflows.

## Anatomy

### Surface
| Property | Value | Token |
|----------|-------|-------|
| Background | `#ffffff` | `bg-surface` |
| Border radius | `16px` | `radius-xl` |
| Shadow | shadow-600 (soft 20px drop + inset hairline) | — |
| Backdrop | `rgba(0,0,0,0.5)` | — |

### Header
| Property | Value | Token |
|----------|-------|-------|
| Background | `#f3f3f3` | `bg-surface-active` |
| Border-bottom | `1px` `#e0e0e0` | `border-default` |
| Padding | `12px 12px 12px 16px` | — |
| Title | 14px / 650 / 20px line-height, `#303030` | `text-default` |
| Close button | 32px, `#616161` icon, `rgba(0,0,0,0.06)` hover | `text-subdued` |

### Body & footer
| Part | Value | Token |
|------|-------|-------|
| Body padding | `16px` (`bodyPadding`), scrolls on overflow | — |
| Footer background | `#ffffff` | `bg-surface` |
| Footer border-top | `1px` `#e0e0e0` | `border-default` |
| Footer | right-aligned, `8px` gap, `16px` padding | — |

## Props

| Prop | Type | Default | Notes |
|------|------|---------|-------|
| `open` | bool | — | Controls visibility. Renders nothing when `false`. |
| `onClose` | fn | — | Called on Escape, backdrop click, and close button. |
| `children` | node | — | Body content (structured) or whole surface (raw). |
| `title` | node | — | Header title. Presence switches Modal into structured mode. |
| `footer` | node | — | Footer action row. Presence switches Modal into structured mode. |
| `size` | `'small' \| 'large' \| 'fullscreen'` | `'large'` | Width token. `fullscreen` fills the viewport. |
| `maxWidth` | number / string | — | Explicit width override; wins over `size`. |
| `bodyPadding` | number | `16` | Padding of the scrollable body region. |
| `closeOnBackdrop` | bool | `true` | Dismiss on scrim click. |
| `closeOnEscape` | bool | `true` | Dismiss on Escape. |
| `showCloseButton` | bool | `true` | Render the header close button. |
| `ariaLabel` | string | — | Accessible name when there's no `title`. |
| `ariaLabelledby` | string | — | ID of an external label element. |
| `zIndex` | number | `1000` | Stacking order of the portal layer. |

## React

```jsx
import { Modal } from 'components/Modal/Modal.jsx';
import { Btn } from 'components/Btn/Btn.jsx';

function ConfirmDelete({ open, onClose, onConfirm }) {
  return (
    <Modal
      open={open}
      onClose={onClose}
      size="small"
      title="Delete reading"
      footer={
        <>
          <Btn variant="secondary" onClick={onClose}>Cancel</Btn>
          <Btn variant="destructive" onClick={onConfirm}>Delete</Btn>
        </>
      }
    >
      This action can't be undone.
    </Modal>
  );
}
```

The footer reuses the shared `Btn` (secondary + primary/destructive), matching the design-system action conventions.
