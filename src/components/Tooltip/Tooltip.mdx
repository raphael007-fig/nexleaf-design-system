# Tooltip

Floating label that briefly explains the function of a UI element. Triggered on hover or focus. Use when an icon or control needs a short text description that would clutter the interface if shown inline.

<Canvas of={TooltipStories.AllStates} />

---

## When to use

- Icon-only buttons that need a label (e.g. filter, export, settings)
- Truncated text that should show the full value on hover
- Form fields with supplementary hints that don't fit as `helpText`

Do **not** use tooltips for critical information — use a `Banner` or inline `helpText` instead. Tooltips are invisible until hovered and must never contain interactive content.

## Two positions

| `position` | Tooltip placement | Arrow direction |
|------------|------------------|-----------------|
| `above` *(default)* | Above the trigger | Points down ↓ |
| `below` | Below the trigger | Points up ↑ |

## Accessibility

- The floating tooltip div uses `role="tooltip"`
- Trigger must have `aria-describedby` pointing to the tooltip in production use
- Tooltip appears on both `:hover` and `:focus` so keyboard users can trigger it
- Never put interactive elements (links, buttons) inside a tooltip

## Angular component

```ts
import { NxTooltipModule } from 'components/Tooltip/tooltip.module';

// Default (above)
<nx-tooltip content="Create new view">
  <button>Add view</button>
</nx-tooltip>

// Below
<nx-tooltip content="Filter records" position="below">
  <nx-btn variant="secondary">Filter</nx-btn>
</nx-tooltip>
```
