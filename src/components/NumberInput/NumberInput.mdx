# NumberInput

Numeric input field. Uses the browser's native number spinner for incrementing. Matches the exact height and style of TextInput — prefix and suffix supported.

<Canvas of={NumberInputStories.AllStates} />

---

## When to use

- **NumberInput** — numeric values where step increments are meaningful (temperature, quantity, weight)
- **TextInput `type="number"`** — plain numeric entry without stepper controls
- **SelectInput** — when the value must come from a predefined list, not a free number

## States

| State | Border | Background | Notes |
|-------|--------|------------|-------|
| Default | `0.66px solid #8a8a8a` | `#fdfdfd` | — |
| Hover | `0.66px solid #616161` | `#fafafa` | — |
| Focus | `0.66px solid #616161` | `#fafafa` | + `0 0 0 2px #005bd3` ring |
| Error | `0.66px solid #8e1f0b` | `#fee9e8` | No focus ring |
| Disabled | none | `rgba(0,0,0,0.08)` | Stepper buttons hidden, `cursor: not-allowed` |
| Read-only | none | `rgba(0,0,0,0.08)` | Stepper buttons hidden, in tab order |

Native browser spinner handles increment/decrement — no custom buttons. Field height is identical to TextInput.

```html

<div class="nx-field">
  <label class="nx-field__label">Quantity</label>

  <label class="nx-field__label">Temperature</label>
  <div class="nx-input-wrap">

    <input class="nx-input" type="number" step="0.1" />
    <span class="nx-input__suffix">°C</span>
  </div>
</div>

<!-- Error state -->
<div class="nx-field">
  <label class="nx-field__label">Temperature</label>
  <div class="nx-input-wrap nx-input-wrap--error">
    <input class="nx-input" type="number" />
  </div>
  <div class="nx-field__error">
    <!-- IcoErrorCircle SVG -->
    Temperature is required.
  </div>
</div>

<!-- Disabled -->
<div class="nx-field">
  <label class="nx-field__label nx-field__label--disabled">Record count</label>
  <div class="nx-input-wrap nx-input-wrap--disabled">
    <input class="nx-input" type="number" value="42" disabled />
  </div>
</div>
```

## Accessibility

- Always pair with a visible `<label>` — never use `placeholder` as the sole label
- Uses the browser's native number spinner — keyboard arrow up/down increments by `step`
- `min` and `max` are enforced natively by the browser
- Required fields use the native `required` attribute on the `<input>`

## Angular component

```ts
import { NxNumberInputModule } from 'components/NumberInput/number-input.module';

// Basic
<nx-number-input label="Quantity" [value]="v" (change)="fn($event)" />

// With suffix and step
<nx-number-input
  label="Temperature"
  suffix="°C"
  [step]="0.1"
  [value]="v"
  (change)="fn($event)"
  helpText="Normal range: 2 – 8 °C"
/>

// With min / max
<nx-number-input
  label="Percentage"
  suffix="%"
  [min]="0"
  [max]="100"
  [value]="v"
  (change)="fn($event)"
/>

// With prefix + suffix
<nx-number-input label="Weight" prefix="~" suffix="kg" [value]="v" (change)="fn($event)" />

// Error state
<nx-number-input
  label="Temperature"
  suffix="°C"
  [required]="true"
  [value]="v"
  (change)="fn($event)"
  error="Temperature is required."
/>

// Disabled
<nx-number-input label="Record count" [value]="42" [disabled]="true" />
```
