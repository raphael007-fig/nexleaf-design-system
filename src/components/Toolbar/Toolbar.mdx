# Toolbar

The application-level top bar. Anchors the logo, breadcrumbs, a global search / AI chat input, region selector, action icon buttons, and the user avatar. Always present at the top of an authenticated screen.

<Canvas of={ToolbarStories.FullToolbar} />

---

## Anatomy

The Toolbar is a stateless **three-column grid** (`1fr · auto · 1fr`) so the center region stays optically centered no matter how wide the start/end clusters grow. An optional fixed-width reserved column can be prepended for app shells that dock a Side Navigation beside the toolbar.

| Region | Typical contents |
|--------|------------------|
| Reserved (optional) | Empty slot under the docked Side Navigation. Enable with `sideNavWidth`. |
| `start` | `<Breadcrumbs />` — or brand logo + `<Breadcrumbs />` if no side nav |
| `center` | `<ToolbarAiChatButton />` — opens `<AiChatPanel />` on the right |
| `end` | `<ToolbarRegionSelector />`, `<ToolbarIconButton />`s, `<ToolbarAvatar />` |

The Toolbar itself is a pure layout primitive — every node inside is composed by the consumer. Subcomponents are exported so each slot's contents stay consistent across screens.

### App shells with Side Navigation

Pair the toolbar with the new `<SideNavigation />` rail by passing `sideNavWidth` equal to the rail's expanded width (default 240 px). The toolbar reserves an empty cell at its left edge so the docked rail visually sits *under* the toolbar — both surfaces share the `#f1f1f1` background, so the seam is invisible.

```jsx
<Toolbar
  sideNavWidth={240}
  start={<Breadcrumbs items={trail} />}
  center={<ToolbarAiChatButton ... />}
  end={<>...</>}
/>
```

The brand logo moves into the Side Navigation in this layout — don't render it in the toolbar's `start` slot too.

## AI Chat Bot button → right-side panel

`<ToolbarAiChatButton />` is a single pill-shaped button (not a text input). Clicking it should open `<AiChatPanel />`, a right-anchored drawer that slides in from the edge of the viewport. The consumer owns the `open` state and the chat content; the design system only ships the chrome.

```jsx
const [open, setOpen] = useState(false);

<Toolbar
  center={
    <ToolbarAiChatButton
      onClick={() => setOpen(true)}
      active={open}
      trailingIcon={<IcoTranslate />}
      onTrailingClick={openTranslate}
    />
  }
/>
<AiChatPanel open={open} onClose={() => setOpen(false)}>
  {/* your chat conversation UI */}
</AiChatPanel>
```

The button exposes `aria-haspopup="dialog"` and `aria-expanded={active}`. Pass `active={open}` so the button visually persists its "open" state (focus ring + halo) while the panel is shown.

## When to use

- Persistent navigation chrome at the top of every authenticated page
- A single place for global, cross-page controls (region, notifications, profile)
- Pair with `<Breadcrumbs />` so the user always sees their location in the trail

Do **not** use the Toolbar for page-specific actions (Save, Delete, Filter). Those belong inside the page body, not the global chrome.

## Mobile responsive

Below ~480px the toolbar collapses to three compact slots: brand mark, **Ask AI** pill, hamburger menu. Pass `mobile` on `<Toolbar>` to tighten the padding and gap; swap in the mobile-tuned subcomponents:

```jsx
<Toolbar
  mobile
  start={<NexleafBrandLogo size={32} />}
  center={
    <ToolbarAskAiButton
      label="Ask AI"
      onClick={() => setChatOpen(true)}
      active={chatOpen}
    />
  }
  end={
    <ToolbarMenuButton
      onClick={() => setMenuOpen(true)}
      expanded={menuOpen}
    />
  }
/>
```

| Desktop subcomponent | Mobile replacement |
|----------------------|--------------------|
| `NexleafLogo` (AI mark) | `NexleafBrandLogo` (leaf + phone) |
| `Breadcrumbs` | — hidden (recover trail via the page header below the toolbar) |
| `ToolbarAiChatButton` ("AI Chat Bot (beta)" + translate) | `ToolbarAskAiButton` ("Ask AI" + chevron) |
| `ToolbarRegionSelector` + icon cluster + avatar | `ToolbarMenuButton` (hamburger → drawer) |

Both AI buttons open the same `<AiChatPanel />` — the drawer behavior is identical. `ToolbarMenuButton` exposes `aria-haspopup="menu"` + `aria-expanded`; wire it to the consumer's own navigation drawer.

## Loading

Pass `loading` on `<Toolbar>` to render the three-region layout with skeleton blocks in place of the real interactive children. Use this on first paint of an authenticated shell, before user/region/notifications data has arrived. The skeleton preserves the toolbar's overall width and height so the page below doesn't reflow when the real toolbar swaps in.

```jsx
<Toolbar loading />
<Toolbar loading mobile />
```

This is a top-level convenience only — individual subcomponents (`ToolbarRegionSelector`, `ToolbarAvatar`, etc.) don't ship their own loading states. If you need a partial skeleton (e.g. just the avatar pending), compose your own `<Skeleton>` blocks in the relevant slot.

Modifiers:

- `.nx-toolbar--elevated` — adds shadow
- `.nx-toolbar--sticky` — `position: sticky; top: 0`
- `.nx-toolbar-iconbtn[disabled]` / `.nx-toolbar-region[disabled]` — disabled states
- `.nx-toolbar-aichat[aria-expanded="true"]` — chat panel is open (persistent focus ring)
- `.nx-aichat-panel__sheet` slides in via `@keyframes slideInRight`

## Accessibility

- Root element is `<header role="banner">` — there should be **one banner per page**
- `<ToolbarAiChatButton>` is a real `<button>` with `aria-haspopup="dialog"` and `aria-expanded={open}` reflecting the panel state. The button label is announced as **"AI Chat Bot (beta)"**
- `<AiChatPanel>` is `role="dialog"` with `aria-modal="true"`; Escape and backdrop click close it. Move focus into the panel on open and restore it to the trigger on close
- `<ToolbarRegionSelector>` has two modes:
  - **Read-only (default).** Renders just the flag + country name with no chevron, no menu, no pointer cursor. This is the right rendering for most users — they can see which region the app is bound to, but not change it. Pass just `countryCode="KE"` + `value="Kenya"`.
  - **Pickable (admin / multi-region operators).** Pass `pickable`, a `countries` array, and an `onCountryChange` handler. The pill grows the disclosure chevron, `aria-haspopup="listbox"` is exposed, and clicking opens a portal `OptionList` of countries (each row renders the same flag glyph as the pill). The `countries` array can come straight from `foundation/emojis/emojiCatalog` so both surfaces share one source of truth.
- The flag glyph is `aria-hidden`; the country name in `value` carries the meaning for assistive tech. Rendering uses the Apple Color Emoji / Segoe UI Emoji / Noto Color Emoji font stack so the platform-native glyph shows up rather than the regional-indicator letter pair.
- Every `<ToolbarIconButton>` **requires** an `ariaLabel` — there is no visible text
- The notification `badge` is decorative; the count must also be reflected in the `ariaLabel` (e.g. `ariaLabel="Notifications, 3 unread"`)
- Focus ring: 2px solid `#005bd3` on every interactive child (matches the design system focus token)
- Keyboard: Tab traverses **start → center → end** in DOM order. Avatar/buttons activate on Enter/Space; the search input accepts text

## Angular component

```ts
import { NxToolbarModule } from 'components/Toolbar/toolbar.module';

<nx-toolbar [sticky]="true" [elevated]="true">
  <ng-container nxToolbarStart>
    <nx-logo></nx-logo>
    <nx-breadcrumbs
      [items]="trail"
      (navigate)="onNavigate($event)">
    </nx-breadcrumbs>
  </ng-container>

  <ng-container nxToolbarCenter>
    <nx-toolbar-aichat
      label="AI Chat Bot"
      [beta]="true"
      [active]="chatOpen"
      [trailingIcon]="'translate'"
      (trailingClick)="onTranslate()"
      (click)="chatOpen = true">
    </nx-toolbar-aichat>
  </ng-container>

  <ng-container nxToolbarEnd>
    <nx-toolbar-region
      [value]="region"
      (click)="openRegionPicker()">
    </nx-toolbar-region>
    <nx-toolbar-iconbtn icon="grid"  ariaLabel="Apps"          (click)="openApps()"></nx-toolbar-iconbtn>
    <nx-toolbar-iconbtn icon="bell"  ariaLabel="Notifications" [badge]="3" (click)="openNotifs()"></nx-toolbar-iconbtn>
    <nx-toolbar-avatar [src]="user.avatar" [alt]="user.name" (click)="openProfile()"></nx-toolbar-avatar>
  </ng-container>
</nx-toolbar>

<!-- Right-side chat drawer -->
<nx-aichat-panel
  [open]="chatOpen"
  title="AI Chat Bot"
  [beta]="true"
  (close)="chatOpen = false">
  <!-- consumer-owned chat conversation UI -->
</nx-aichat-panel>
```

## React component

```jsx
import {
  Toolbar,
  ToolbarAiChatButton,
  ToolbarRegionSelector,
  ToolbarIconButton,
  ToolbarAvatar,
  AiChatPanel,
  NexleafLogo,
} from 'components/Toolbar/Toolbar.jsx';
import { Breadcrumbs } from 'components/Breadcrumbs/Breadcrumbs.jsx';

const [trail, setTrail] = useState([
  { label: 'Home' },
  { label: 'Equipment Management' },
  { label: 'ColdChain Equipment' },
]);
const [chatOpen, setChatOpen] = useState(false);

<>
  <Toolbar
    sticky
    elevated
    start={
      <>
        <NexleafLogo size={32} />
        <Breadcrumbs items={trail} onSelect={(_id, _item, i) => setTrail(trail.slice(0, i + 1))} />
      </>
    }
    center={
      <ToolbarAiChatButton
        label="AI Chat Bot"
        beta
        active={chatOpen}
        trailingIcon={<IcoTranslate />}
        onTrailingClick={() => openTranslate()}
        onClick={() => setChatOpen(true)}
      />
    }
    end={
      <>
        <ToolbarRegionSelector countryCode="KE" value="Kenya" />
        <ToolbarIconButton icon={<IcoGrid />} ariaLabel="Apps" onClick={() => openApps()} />
        <ToolbarIconButton icon={<IcoBell />} ariaLabel="Notifications, 3 unread" badge={3} onClick={() => openNotifs()} />
        <ToolbarAvatar src={user.avatar} alt={user.name} onClick={() => openProfile()} />
      </>
    }
  />

  <AiChatPanel open={chatOpen} onClose={() => setChatOpen(false)}>
    {/* Your chat conversation UI */}
  </AiChatPanel>
</>
```
