# Tag

A pill-shaped chip used to represent a keyword, selected value, or applied filter. Used inside multi-select inputs (`SearchSelectMulti`), filter bars, and AI-suggestion lists.

<Canvas of={TagStories.AllStates} />

---

## When to use

- **Tag** — represent a user-selected value (filter chip, multi-select chip, keyword)
- **Badge** — represent system-supplied status or category (no user action)
- **Btn (small)** — a primary interaction the user is being asked to perform

Tags should never be the primary action on a screen. If a user clicks a tag, it should either filter, remove, or open a detail — never submit a form.

## Accessibility

- The X button is a real `<button>` with `aria-label="Remove {label}"` — screen readers announce both label and action
- Clickable tags (via `onClick`) are focusable (`tabIndex={0}`) and respond to Space / Enter
- Removable tags also respond to **Backspace** / **Delete** when focused
- Disabled tags have `tabIndex={-1}` and `cursor: not-allowed`; their X button is hidden via `disabled`
- The wrapper exposes a visible focus ring (`0 0 0 2px #005bd3`) — never override with `outline: none` without a replacement
- Icon glyphs are decorative — wrapped in `aria-hidden="true"`. Never use icon as the only label.

```ts
import { NxTagModule } from 'components/Tag/tag.module';

// Basic

// Removable
<nx-tag label="Africa" removable (remove)="removeRegion('Africa')" />

<nx-tag label="Smart suggestion" tone="magic" [icon]="infoIcon" removable (remove)="dismiss()" />

// Group
<nx-tag-group>
  <nx-tag *ngFor="let t of tags" [label]="t" removable (remove)="remove(t)" />

```

## React component

```jsx
import { Tag, TagGroup } from 'components/Tag/Tag.jsx';

// Basic
<Tag label="Africa" />

// Removable (used inside SearchSelectMulti)
<Tag label="Africa" removable onRemove={() => removeRegion('Africa')} />

// Clickable filter
<Tag label="Active" onClick={() => toggleFilter('active')} />

// Magic tone (AI-suggested chip)
<Tag label="Predictive" tone="magic" icon={<IcoInfo />} removable onRemove={dismiss} />

// Disabled
<Tag label="Locked" disabled />

// Group of tags (auto-wrap)
<TagGroup>
  {tags.map(t => (
    <Tag key={t} label={t} removable onRemove={() => remove(t)} />
  ))}
</TagGroup>
```

**Props**

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `label` | `string` | — | Tag text (also accepts `children`) |
| `tone` | `'default' \| 'magic'` | `'default'` | Color treatment |
| `icon` | `ReactNode` | — | Optional leading icon, 14×14 |
| `removable` | `boolean` | `false` | Show trailing X button |
| `onRemove` | `() => void` | — | Fires when X is clicked or Backspace/Delete pressed |
| `onClick` | `(e) => void` | — | Makes the tag clickable + focusable |
| `disabled` | `boolean` | `false` | Disables interaction; hides X |

## Usage inside SearchSelect

`SearchSelectMulti` renders selected values as a `<TagGroup>` of removable Tags below the field. When a chip's X is clicked, the multi-select removes that value from `value` and calls `onChange` with the new array.

```jsx
<SearchSelectMulti
  label="Regions"
  options={REGIONS}
  value={regions}
  onChange={setRegions}
/>
// internally renders:
// <TagGroup>
//   {regions.map(r => <Tag key={r} label={labelOf(r)} removable onRemove={() => remove(r)} />)}
// </TagGroup>
```
