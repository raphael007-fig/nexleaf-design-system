# TextInput

Single-line text entry field. Used for short free-form values: names, temperatures, IDs, search queries.

<Canvas of={TextInputStories.AllStates} />

---

## When to use

- **TextInput** — short free-form string, number, email, password, search
- **TextareaInput** — multi-line text (comments, notes, long descriptions)
- **SelectInput** — when the user must choose from a fixed list
- **NumberInput** — when you need numeric step controls

## Prefix & Suffix

Use `prefix` and `suffix` to attach context to either side of the input value. Both accept a string (text label) or a JSX element (inline SVG icon). They can be used independently or combined.

<Canvas of={TextInputStories.PrefixAndSuffix} />

| Prop | Type | Example |
|------|------|---------|
| `prefix` | `string \| ReactNode` | `"$"` · `"~"` · `<SearchIcon />` |
| `suffix` | `string \| ReactNode` | `"°C"` · `"lbs"` · `"USD"` · `<CalendarIcon />` |

  <div class="nx-field__label"><span>Temperature</span></div>

<div class="nx-field">
  <div class="nx-field__label"><span>Email</span></div>
  <div class="nx-input-wrap nx-input-wrap--error">

  </div>
  <div class="nx-field__error">

  </div>
</div>

  <div class="nx-field__label"><span>Record ID</span></div>

  </div>

<div class="nx-field">
  <div class="nx-field__label"><span>Password</span></div>

## Accessibility

- Always pair inputs with a visible `<label>` — never use `placeholder` as the only label
- Required fields use the `required` attribute (not just the visual asterisk)
- Error messages must be connected via `aria-describedby` pointing to the error element
- Disabled inputs use the native `disabled` attribute — assistive technology announces state
- `readonly` inputs behave like disabled visually but remain in the tab order and are announced differently

## Angular component

```ts
import { NxTextInputModule } from 'components/TextInput/text-input.module';

// Basic
<nx-text-input label="Store name" placeholder="e.g. My Store" [value]="v" (change)="fn($event)" />

// Required with suffix
<nx-text-input label="Temperature" [required]="true" suffix="°C" type="number" [value]="v" (change)="fn($event)" />

// Error state
<nx-text-input label="Email" [value]="v" (change)="fn($event)" error="Enter a valid email address." />

// Disabled
<nx-text-input label="Record ID" value="REC-0042" [disabled]="true" />

// With help text
<nx-text-input label="Password" type="password" helpText="Must be at least 8 characters." [value]="v" (change)="fn($event)" />

// Clear button
<nx-text-input label="Search" [clearButton]="true" [value]="v" (change)="fn($event)" />
```

## React component

```jsx
import { TextInput } from 'components/TextInput/TextInput.jsx';

// Basic
<TextInput label="Store name" placeholder="e.g. My Store" value={v} onChange={fn} />

// Required with suffix
<TextInput label="Temperature" required suffix="°C" type="number" value={v} onChange={fn} />

// Error state
<TextInput label="Email" value={v} onChange={fn} error="Enter a valid email address." />

// Disabled
<TextInput label="Record ID" value="REC-0042" disabled />

// With help text
<TextInput label="Password" type="password" helpText="Must be at least 8 characters." value={v} onChange={fn} />

// Clear button (appears when field has value)
<TextInput label="Search" clearButton value={v} onChange={fn} />

// Magic / AI tone
<TextInput label="AI prompt" tone="magic" placeholder="Describe your product..." value={v} onChange={fn} />
```
